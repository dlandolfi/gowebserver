Encryption!

For whatever reason, the names Alice and Bob are used a lot when explaining encryption, so I'm going to carry on that tradition.

Say Alice wants to mail a letter to Bob, but she doesn't want anyone to be able to read it at any point during its transit. What she can do is encrypt the message with a cipher. The first known cipher is known as the Caesar cipher, named after Julius Caesar. This involves shifting the alphabet by a certain number. For this to work, both the sender and the recipient need to share that number to encrypt and decrypt the message. This number is known as an encryption key.

Let's imagine the key is "5" and we're shifting 5 spaces forward in the alphabet. Then A's become F's, B's become G's, C's become H's, and so on. So the message "Hello Bob" becomes "Mjqqt Gtg". Now, anyone intercepting this message will be reading some encrypted nonsense. And for Bob, all he has to do is shift by 5 in the opposite direction to decrypt the message and be able to read it. This is a lot like how the messaging app Signal works. If you click on a contact and tap Verify Safety Number, you'll see the key that is used to encrypt/decrypt messages between you and that person. Every one of your contacts will have a different key.

In a similar way this works with websites, where the browser on your device and a web server are the sender and receiver. In a very broad sense, when you visit a secure website, that server sends you the key and then it sends you the contents of the website in encrypted form for your browser to decrypt once it arrives on your device. Since you both have the key you can send encrypted info back and forth. This is mostly important when sensitive information is being transferred so nothing can be revealed during transit. It isn't at all necessary for this site, but I wanted to know how to implement it anyway.

This is a pretty vague description but I think it paints pretty good picture of what's happening. And this is also not perfectly secure. No encryption is perfectly secure. If you'd like to know more about this kind of thing I highly recommend The Code Book by Simon Singh. It is an easy to digest read that is really fun! :D
